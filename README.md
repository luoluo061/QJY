# 电商用户流失预测项目

这是一个围绕电商用户行为日志构建的用户流失预测项目。项目目标是把原始行为流水整理为用户级特征，完成时间窗切分、标签构建、模型训练、模型优化与结果可视化，形成一套可重复运行的建模流程。

## 项目定位

- 输入：小时级用户行为日志
- 输出：用户级建模数据、模型对比结果、特征分析结果、可汇报图表
- 问题定义：根据观察期行为，预测用户在后续预测期内是否完全停止活跃

当前实现采用固定时间窗：

- 观察期：`2014-11-18 00:00:00` 到 `2014-12-11 23:00:00`
- 预测期：`2014-12-12 00:00:00` 到 `2014-12-18 23:00:00`

标签定义如下：

- 预测期内仍有行为：`label_churn = 0`
- 预测期内无任何行为：`label_churn = 1`

## 开发思路

这个项目的核心不是单独堆模型，而是先把建模链路做完整，再比较不同算法在同一口径下的表现。

整体思路分成五步：

1. 先审计原始数据，确认字段、时间范围、缺失情况和行为分布。
2. 再按时间顺序划分观察期和预测期，避免把未来信息带入特征。
3. 然后把行为流水聚合为用户级特征，构建活跃度、近期行为、行为结构和兴趣广度等变量。
4. 在统一训练测试划分上比较 baseline 模型。
5. 最后做特征筛选、类别不平衡处理和参数搜索，输出优化后的结果与解释图。

这里面最关键的设计点有三个：

- 先按时间顺序定义预测任务，而不是随机打标签
- 先把行为日志聚合成用户画像，再交给分类模型训练
- 先建立 baseline，再做优化，保证实验对比有可解释的基线

## 技术方案

### 数据处理

- 使用 `pandas` 按块读取大体量 CSV，降低内存占用
- 使用 `Counter`、`defaultdict` 和集合结构累计用户级统计量
- 通过显式时间窗控制特征与标签的边界，避免数据泄漏

### 特征工程

当前特征主要来自四类信息：

- 活跃度：总行为数、活跃天数、活跃小时数、活跃跨度
- 近期行为：最近 7 天行为数、最近 3 天行为数、最近活跃距窗口结束时间
- 行为结构：各行为类型次数与占比
- 兴趣广度：交互商品数、交互类目数及相关比值

这些特征背后的设计逻辑是：

- 用总量特征描述用户整体活跃程度
- 用近期特征描述用户是否开始降温
- 用结构特征描述用户行为习惯是否变化
- 用广度特征描述用户兴趣面是否收缩

### 建模与优化

项目当前对比了四类模型：

- `LR`
- `RF`
- `XGBoost`
- `LightGBM`

优化阶段包含三件事：

- 基于互信息的特征筛选
- 面向类别不平衡的权重处理
- 基于 `RandomizedSearchCV` 的参数搜索

评价指标以 `ROC_AUC`、`F1`、`Precision`、`Recall` 为主，同时补充 ROC 曲线、混淆矩阵和特征重要性分析。

## 实验路径

这个项目的实验路径可以概括为：

### 第一阶段：确认数据可用

- 检查时间范围是否连续
- 检查缺失值是否可控
- 检查行为类型分布是否合理

### 第二阶段：构造预测任务

- 先定义观察期
- 再定义预测期
- 用预测期内是否仍活跃来定义流失标签

### 第三阶段：构造用户画像

- 统计活跃天数、活跃小时数、总行为数
- 统计最近 7 天、最近 3 天行为量
- 统计各行为类型出现次数和占比
- 统计商品与类目覆盖广度

### 第四阶段：建立基线模型

- 先在同一训练测试划分下比较四种模型
- 观察不做复杂优化时哪类模型已经具备较强区分能力

### 第五阶段：做优化实验

- 用互信息筛掉噪声特征
- 对不平衡样本做类别权重修正
- 用随机搜索找更合适的超参数组合

### 第六阶段：做结果解释

- 看 ROC / PR 曲线
- 看混淆矩阵
- 看特征重要性
- 看扩展解释图

## 当前结果摘要

基于现有数据口径，当前结果表现为：

- Baseline 阶段按 `ROC_AUC` 最优模型为 `LR`
- 优化阶段按 `ROC_AUC` 最优模型为 `XGBoost`
- 优化阶段按 `F1` 最优模型为 `RF`

说明这个任务中：

- 线性可分信号是存在的
- 树模型在优化后对复杂非线性关系的利用更充分
- 在类别不平衡前提下，不同指标会对应不同“最优模型”

## 如果老师顺着代码往下问

比较容易被问到的问题通常有这些：

### 为什么不能直接用原始日志训练

因为原始日志是一条行为一行，而分类模型需要稳定的样本粒度。这里先统一到“每个用户一行”，才能形成可训练的监督学习数据集。

### 为什么要切观察期和预测期

因为流失预测本质上是时间相关任务。特征必须来自过去，标签必须来自未来，否则会引入数据泄漏。

### 为什么要同时比较四种模型

因为线性模型、Bagging 模型和 Boosting 模型在表格数据上的归纳方式不同，比较后才能判断这个任务更适合哪类模型。

### 为什么不能只看准确率

因为流失样本占比低，准确率容易被多数类掩盖，所以必须结合 `Recall`、`F1`、PR 曲线一起看。

### 为什么优化后树模型更好

因为树模型对非线性关系和特征交互的利用更强，在筛掉噪声特征并处理不平衡之后，优势更容易体现出来。
